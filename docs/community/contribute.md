# 贡献

如果你准备反馈问题、补正文档或提交代码，建议按下面的顺序进行。

## 先按事情找

| 场景 | 建议 |
| --- | --- |
| 可复现的 Bug、缺失说明或新需求 | 提交 Issue |
| 小范围文档修正或小范围代码修复 | 直接提交 PR |
| 改动较大、方向还不明确 | 先开 Issue 或先到 [交流](/community/discuss.md) 说明 |

## 提交问题前先做什么

1. 先看 [FAQ](/reference/faq.md)
2. 再看 [补充说明](/reference/notes.md)
3. 确认是否已经有相同或相似的 [GitHub Issues](https://github.com/djylb/nps/issues)

## 适合提交 Issue 的内容

- 明确可复现的 Bug
- 文档错误、说明缺失或位置错位
- 新需求或改进建议
- 平台兼容问题

提交时尽量附带：

- NPS 和 NPC 版本
- 操作系统和部署方式
- 关键配置片段
- 复现步骤
- 相关日志

如果是文档问题，建议额外附带：

- 出问题的页面路径
- 哪一段内容不准确或不清楚
- 你期望的写法或补充点

## 适合直接提交 PR 的内容

- 文档修正
- 小范围 Bug 修复
- 测试补充
- 不改变现有行为的整理性改动

代码贡献建议：

1. Fork 仓库
2. 在独立分支上修改
3. 保持改动范围清晰
4. 提交 PR 并说明变更目的、影响范围和验证方式

如果你修改的是文档，建议同时检查：

- 页面放在的分类是否合适
- 链接是否仍然正确
- 表述是否容易直接翻译
- 示例里的代码、字段名和路径是否使用英文

## 讨论新功能时的建议

如果改动较大，建议先开 Issue 或在 [交流](/community/discuss.md) 中说明目标和使用场景，再决定是否直接提交 PR。
