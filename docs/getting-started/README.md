# 开始使用

这一部分只面向第一次部署和第一次连通。先完成最小链路，再进入操作指南或参考资料。

## 推荐顺序

| 顺序 | 页面 | 用途 |
| --- | --- | --- |
| 1 | [10 分钟快速开始](/getting-started/quick-start.md) | 用最短路径完成一次验证 |
| 2 | [安装指南](/getting-started/install.md) | 按环境选择 Docker、脚本、发布包或源码 |
| 3 | [启动 NPS 服务端](/getting-started/start-server.md) | 启动服务端并登录 Web 管理界面 |
| 4 | [启动 NPC 客户端](/getting-started/start-client.md) | 复制 Web 页面里的客户端命令并连上服务端 |
| 5 | [首次选择隧道类型](/getting-started/tunnel-types.md) | 确认应该先用哪类规则 |
| 6 | [架构与核心概念](/getting-started/architecture.md) | 补齐基本概念 |

第一次启动 NPC 时，优先使用 Web 管理界面“客户端列表”提供的快捷命令。确认命令行连接正常后，再考虑服务、自启动、配置文件或 `-launch`。
