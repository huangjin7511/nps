# 开始使用

这一部分面向第一次部署 NPS 的用户，按“安装、启动、验证、理解基本概念”的顺序组织。

这里的目标很明确：

- 帮助你完成第一次部署
- 帮助你完成第一次连接验证
- 帮助你知道下一步应该进入哪一类文档

这里不展开精确配置字段、接口字段或完整协议规范。需要这些内容时，请分别进入 [参考资料](/reference/README) 或 [接口与集成](/reference/integration/README)。

## 最短阅读路径

如果你只想先完成部署和验证，按下面顺序阅读即可：

1. [10 分钟快速开始](/getting-started/quick-start)
2. [安装指南](/getting-started/install)
3. [启动 NPS 服务端](/getting-started/start-server)
4. [启动 NPC 客户端](/getting-started/start-client)

第一次启动客户端时，建议优先在 Web 管理界面的“客户端列表”复制快捷启动命令，先完成命令行连接验证，再决定是否使用服务、自启动或 `npc.conf`。

## 进入条件

完成这一部分后，你通常已经具备以下条件：

- 一台可从公网访问的服务器
- 一个可以从内网访问目标服务的客户端环境
- 一条已经验证可用的最小规则

## 之后去哪里

- 需要日常操作步骤：进入 [操作指南](/guide/README)
- 需要选择规则类型：进入 [选型与规则](/guide/design/README)
- 需要精确配置字段：进入 [参考资料](/reference/README)
- 需要接口或平台接入：进入 [接口与集成](/reference/integration/README)
