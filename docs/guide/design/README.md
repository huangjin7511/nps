# 选型与规则

这一部分专门处理“应该选哪一种规则”和“选定后去哪里看详细说明”这两个问题。

把“选型”单独放成一类，有两个作用：

1. 新用户可以先完成选择，再进入具体配置页面。
2. 后续新增规则类型时，只需要同时更新这一类入口页和对应的说明页，不需要把选择逻辑散落到多个操作页中。

## 建议从哪一页进入

| 你当前的问题 | 建议页面 |
| --- | --- |
| 我还不清楚应该选哪一类转发方式 | [规则选型总览](/guide/design/tunnel-selection.md) |
| 我已经知道业务目标，但还不确定规则类型 | [场景总览与选型](/guide/scenarios/common.md) |
| 我已经知道规则类型，只想看具体说明 | [隧道与转发类型](/guide/tunnels/README.md) |

## 这一部分包含什么

- 面向第一次选择的总览页
- 面向业务目标的场景页
- 面向具体规则类型的说明页

## 这一部分不包含什么

- 服务启动和运维步骤：看 [操作指南](/guide/README.md)
- 精确配置字段、限制和兼容边界：看 [参考资料](/reference/README.md)
- API、SDK 和启动协议：看 [接口与集成](/reference/integration/README.md)
