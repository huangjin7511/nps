# TCP 隧道

TCP 隧道用于把一个公网 TCP 端口映射到一个内网 TCP 服务。

## 适合什么场景

- SSH
- RDP
- MySQL、PostgreSQL、Redis
- 任意自定义 TCP 协议

## Web 管理界面最常用字段

| 字段 | 作用 |
| --- | --- |
| 监听端口 | 公网访问的端口 |
| 内网目标 | 实际要访问的内网 `IP:PORT` |
| 备注 | 方便管理识别 |

## 最小示例

假设：

- 公网服务器 IP 是 `1.1.1.1`
- 内网主机 `10.1.50.101:22` 提供 SSH
- 你希望通过公网 `8001` 访问它

在客户端下创建一条 TCP 隧道：

- 监听端口：`8001`
- 内网目标：`10.1.50.101:22`

然后从外部访问：

```bash
ssh -p 8001 root@1.1.1.1
```

## `npc.conf` 示例

```ini
[common]
server_addr=1.1.1.1:8024
vkey=123

[tcp]
mode=tcp
target_addr=127.0.0.1:8080
server_port=9001
```

## 常见能力

- 端口白名单
- 端口范围映射
- 连接数限制
- 流量和带宽限制
- Proxy Protocol

## 注意事项

- TCP 隧道只适合 TCP 服务，不适合 UDP
- 如果服务端限制可开放端口，需检查 `allow_ports`
- 如果需要把真实地址传给后端，且后端支持 Proxy Protocol，可以启用相关能力
