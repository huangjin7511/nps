# UDP 隧道

UDP 隧道用于把一个公网 UDP 端口映射到一个内网 UDP 服务。

## 适合什么场景

- DNS
- 游戏联机
- 音视频流
- 任意自定义 UDP 服务

## Web 管理界面最常用字段

| 字段 | 作用 |
| --- | --- |
| 监听端口 | 公网访问的 UDP 端口 |
| 内网目标 | 实际要访问的内网 `IP:PORT` |
| 备注 | 方便管理识别 |

## 最小示例

假设：

- 公网服务器 IP 是 `1.1.1.1`
- 内网 DNS 服务是 `10.1.50.102:53`

创建一条 UDP 隧道：

- 监听端口：`10053`
- 内网目标：`10.1.50.102:53`

之后把外部设备的 DNS 指向 `1.1.1.1:10053` 对应的入口，即可通过 NPS 使用该内网 DNS。

更容易复现的测试方式是直接用带端口的 DNS 工具，例如：

```bash
dig @1.1.1.1 -p 10053 example.com
```

第一次验证更建议使用这种高位端口，而不是直接占用 `53`：

- 更不容易和本机 DNS 服务冲突
- 更不容易受到系统权限限制
- 防火墙放行时更容易确认

只有在你明确需要标准 `53/udp`，并且已经处理好端口占用与防火墙时，再把监听端口改成 `53`。

## `npc.conf` 示例

```ini
[common]
server_addr=1.1.1.1:8024
vkey=123

[udp]
mode=udp
target_addr=127.0.0.1:5353
server_port=9002
```

## 常见能力

- 端口白名单
- 端口范围映射
- 流量和带宽限制
- Proxy Protocol

## 注意事项

- UDP 隧道只适合 UDP 服务
- 一些云平台和本地防火墙默认不会放行 UDP，需要单独检查
- 如果后端需要真实来源地址，只有在后端支持 Proxy Protocol 时才适合开启相关能力
