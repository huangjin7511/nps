# 功能清单：代理、转发与路由

这一页集中放代理转发链路、端口映射、端口复用和目标路由相关能力。

如果你要看站点级 Header、证书或 URL 路由，去看 [站点与 HTTP](/reference/features-http.md)。

## 嵌套转发支持

可在隧道的 **目标** 中配置其他隧道，实现链式转发。

- 转发到隧道：在目标填写 `tunnel://<tunnel-id>`（如 `tunnel://11`）
  - 需开启 **“代理到服务器本地”**
  - **支持类型**：TCP 相关隧道（TCP 隧道、混合代理（HTTP/SOCKS5）、文件隧道等）

- 转发到桥接：在目标填写 `bridge://<bridge-type>`（如 `bridge://tcp`）
  - 需**管理员权限**且开启 **“代理到服务器本地”**
  - **支持类型**：`tcp`、`tls`、`ws`、`wss`、`web`

此外 **文件访问** 隧道还支持下面方式嵌套：

- 转发到文件：在目标填写 `file://<file-uuid>`（从管理页面文件访问列表获取）
  - **不要** 开启 “代理到服务器本地”
  - 发起嵌套的隧道客户端 **必须与该文件隧道所属客户端相同**

## Proxy Protocol

该功能用于在 **TCP 隧道**、**UDP 隧道** 和 **域名转发** 中向后端传递真实来源地址，但要求后端本身支持 Proxy Protocol。

## 负载均衡

域名转发和 TCP 隧道都支持把多个内网目标写在同一条规则中，用于轮询级别的负载均衡。

## 端口范围映射

当客户端以配置文件方式启动时，可以将本地端口做范围映射，仅支持 TCP 和 UDP。例如：

```ini
[tcp]
mode=tcp
server_port=9001-9009,10001,11000-12000
target_port=8001-8009,10002,13000-14000
```

支持单个端口和端口范围，多个值用逗号分隔。

注意：

- `server_port` 和 `target_port` 必须一一对应
- 无法对应的范围不会成功创建

## 端口范围映射到其他机器

```ini
[tcp]
mode=tcp
server_port=9001-9009,10001,11000-12000
target_port=8001-8009,10002,13000-14000
target_ip=10.1.50.2
```

填写 `target_ip` 后，表示把这一组端口映射到另一台机器。
留空时默认映射到本地 `127.0.0.1`。
这一项只在端口范围映射时生效。

## 端口复用

在一些严格的网络环境里，可开放的公网端口数量很少，这时可以把多个入口复用到同一个端口。

常见做法是把 `bridge_tcp_port`、`http_proxy_port`、`https_proxy_port` 和 `web_port` 配到同一端口，再通过请求特征和域名区分访问流量。

- 如果 Web 管理也参与复用，通常需要同时配置 `web_host`
- 如果桥接连接参与复用，通常还要结合 `bridge_host` 或对应连接协议入口一起规划
- 端口复用会提高部署复杂度，第一次部署建议先分开端口，确认链路可用后再合并
